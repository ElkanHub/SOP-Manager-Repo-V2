# SOP-Guard Pro Documentation

This folder is the documentation hub for the project. Versioned product docs live separately so v2.1 material does not get mixed into v2.2 work.

## Start Here

- [v2.2 documentation](v2.2/README.md) - current product, implementation, strategy, operations, and audit docs.
- [v2.1 documentation](v2.1/README.md) - historical v2.1 build and UI specs.
- [Legacy archive](archive/legacy/) - older comparison and blueprint files kept for reference only.

## Folder Rules

- Put current docs under `v2.2/`.
- Put historical v2.1 docs under `v2.1/`.
- Put old or superseded non-versioned material under `archive/`.
- Keep root project files such as `README.md` and `CLAUDE.md` at the repository root.

## Current Structure

- `v2.2/specs/` - canonical build, UI, and documentation specs.
- `v2.2/build-plans/` - implementation plans for specific features.
- `v2.2/strategy/` - product, platform, tenancy, numbering, AI, and process strategy.
- `v2.2/operations/` - progress, routes, testing, PWA, and runtime reference docs.
- `v2.2/audits/` - audit reports, reality checks, and issue lists.
- `v2.2/reference/` - alignment notes and supporting reference material.
- `v2.2/source-documents/` - uploaded source documents, PDFs, and meeting notes.
- `archive/legacy/` - older reference documents that should not drive current builds.
