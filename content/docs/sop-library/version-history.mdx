Every SOP in SOP-Guard Pro keeps its complete history. You can view any previous version at any time.

---

## Opening Version History

1. Open any SOP in the viewer
2. Click **Version History** in the header

The Version History panel opens from the right side.

---

## What Version History Shows

For each version, you see:

- **Version number** — e.g., v1.0, v1.1, v2.0
- **Uploaded by** — Who submitted this version
- **Date** — When it was uploaded
- **Status** — Whether this is the current active version or superseded

The current Active version is highlighted with a "Current" badge.

---

## Viewing Old Versions

Click **View** on any version to open it in the viewer. When viewing an old version:

- A banner appears at the top: **"SUPERSEDED VERSION"**
- It shows which version you are viewing and which is current
- You can click **View current** to switch to the latest version

Old versions are read-only — you can see what the procedure looked like then, but you cannot acknowledge old versions or make changes.

---

## Why Version History Matters

Version history is essential for:

- **Compliance audits** — Prove what procedures were in effect at any point in time
- **Change tracking** — Understand how a procedure evolved over time
- **Reference** — Find procedures that were replaced but might be needed again
