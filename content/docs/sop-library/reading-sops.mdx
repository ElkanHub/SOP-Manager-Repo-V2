Reading SOPs is what most users do most of the time. This guide covers how to navigate and read procedures effectively.

---

## Opening an SOP

1. Go to **SOP Library** in the sidebar
2. Find the SOP you want to read (use the search or scroll)
3. Click anywhere on the row

The SOP opens in a new tab within the Library.

---

## The Viewer Header

When an SOP is open, the header shows important metadata:

- **SOP Number** — The unique identifier (e.g., SOP-001)
- **Title** — The procedure name
- **Department** — The owning department (with badge)
- **Version** — Current version (e.g., v1.0)
- **Status** — Active, Draft, Pending QA, etc.
- **Date Listed** — When it became active
- **Last Revised** — When it was last updated
- **Due for Revision** — When it needs review

---

## Reading the Document

The SOP document itself is rendered from the uploaded Word file. It displays as read-only HTML:

- All formatting is preserved (headings, lists, tables, images)
- The layout is clean and readable
- You can scroll through long documents

<Callout type="info">
  SOPs are written in Microsoft Word and uploaded as .docx files. SOP-Guard Pro does not have an in-app editor — this is intentional. The platform is a governance and distribution system, not a document editor.
</Callout>

---

## Multiple Tabs

You can have multiple SOPs open at once:

- Click the **+** button to open a new tab
- Click any open tab to switch between SOPs
- Click the **×** on a tab to close it
- Up to 8 tabs can be open simultaneously

This is useful when you need to compare procedures or reference one while working on another.

---

## Version History

Click **Version History** to see all previous versions of the SOP. You can view any old version, which is useful for understanding how a procedure evolved over time.

When viewing an old version, a banner appears indicating it is **SUPERSEDED** — you are viewing historical content, not the current operative version.
