When a QA Manager approves an SOP update, a **Change Control** is created. This is a formal process that ensures all affected managers have reviewed and approved the changes before they go live.

---

## What is a Change Control?

A Change Control (CC) is a formal record that:

- Tracks changes to an SOP between versions
- Ensures all affected managers have signed off
- Creates an immutable audit trail

Think of it as a "sign-off sheet" for procedure updates.

---

## When is a CC Created?

A Change Control is created when:

- QA approves an **update** to an existing SOP
- It is NOT created for new SOPs (those go directly to Active)

The CC captures the "old" version and "new" version, shows the differences, and requires manager signatures.

---

## Who Must Sign

The required signatories are determined at CC creation time:

- All **Managers** in the SOP's primary department
- The **QA Manager** who approved the update

This snapshot is taken when the CC is created. If a manager leaves or is transferred mid-CC, they are still required to sign (or an admin must waive their signature).

---

## The 14-Day Deadline

Every Change Control has a **14-day deadline** from creation. All signatures must be collected within this time.

If the deadline passes:
- Admins receive escalation notifications
- The CC shows as overdue
- An admin may need to waive outstanding signatures

---

## What Happens When Complete

When all required signatures are collected:

1. The new SOP version becomes **Active**
2. The old version becomes **Superseded**
3. All department members receive a Pulse notification
4. They must acknowledge the new version
