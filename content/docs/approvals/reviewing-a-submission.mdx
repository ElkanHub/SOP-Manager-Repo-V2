## The Request Modal Interface

When you click to review a submission, a modernized Request Details Modal will open. This modal provides a comprehensive, professional workspace for reviewing the submission.

The interface is divided into two primary sections:

### Main Content Area (Left)
- **Document / Details Viewer:** View the submitted procedure document directly. If the submission involves updates, you may see detailed comparisons or side-by-side diffs.
- **Request Metadata:** Structured cards detailing the request type, reference numbers, and submission contexts.
- **Requester Information:** A clear profile card of the submitter, including their department and contact info.

### Processing Panel (Right)
- **Current Status:** A visual badge indicating the state of the request (e.g., Pending, Changes Requested).
- **Processing Actions:** A unified action strip where you can quickly Approve, Request Changes, or Reject the submission.
- **Notes and History:** Review previous comments, the timeline of the request, and add new notes or feedback directly.

---

## The Review Thread

The review thread shows the history:

- Every time you request changes or approve, it is recorded
- Every resubmission is tracked
- Comments are preserved

This lets you see the full context of the submission.

---

## Starting Your Review

1. Open the submission
2. Read the document in the left panel
3. Check the Notes to QA for context
4. Decide: Approve, Request Changes, or Reject
