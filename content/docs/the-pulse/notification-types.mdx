Every item in the Pulse has a type that determines how it looks, who receives it, and what action (if any) is required.

---

## Notification Types

### approval_request

- **When:** A Manager submits an SOP for QA review
- **Who sees it:** QA Managers
- **What it looks like:** Blue icon, shows the SOP number and title
- **Action required:** Click to open the Approvals page and review the submission

### approval_update

- **When:** QA responds to a submission (approves, requests changes, or rejects)
- **Who sees it:** The Manager who submitted
- **What it looks like:** Blue icon with the response type
- **Action required:** Check the response in Approvals

### cc_signature

- **When:** A Change Control requires your signature
- **Who sees it:** All required signatories (Managers in the SOP's department)
- **What it looks like:** Navy icon with pen symbol
- **Action required:** Click to open the Change Control and sign

### cc_deadline

- **When:** A Change Control is approaching or past its deadline
- **Who sees it:** Admins
- **What it looks like:** Red/amber icon warning of deadline
- **Action required:** Consider waiving a signature if the signatory is unavailable

### pm_due

- **When:** A preventive maintenance task is due today
- **Who sees it:** The assigned person
- **What it looks like:** Amber/blue icon with equipment name
- **Action required:** Complete the task and log it

### pm_overdue

- **When:** A PM task has passed its due date
- **Who sees it:** The assigned person and their manager
- **What it looks like:** Red icon — this is urgent
- **Action required:** Complete the task immediately

### sop_active

- **When:** An SOP has been updated to a new version
- **Who sees it:** Everyone in the SOP's department
- **What it looks like:** Green check icon
- **Action required:** Read the updated SOP and acknowledge it

### notice

- **When:** A broadcast announcement is sent
- **Who sees it:** The target audience (department or everyone)
- **What it looks like:** Grey bell icon
- **Action required:** Read the notice; you can reply inline

### message

- **When:** A new chat message arrives in Messaging
- **Who sees it:** The conversation participants
- **What it looks like:** Chat bubble icon
- **Action required:** Click to open the conversation in Messages

### todo

- **When:** You add a personal to-do
- **Who sees it:** Only you
- **What it looks like:** Purple checklist icon
- **Action required:** Complete the task and check it off

### system

- **When:** Platform-level announcements
- **Who sees it:** All users or specific groups
- **What it looks like:** Grey info icon
- **Action required:** Varies by message

---

<Callout type="tip">
  Priority group items (orange/red dot) always appear at the top of the Pulse regardless of when they arrived. Check the Priority group first — these are the items that can block other people's work if you delay.
</Callout>
