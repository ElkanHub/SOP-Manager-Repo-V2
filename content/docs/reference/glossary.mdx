## Terms

**Acknowledgement** — A formal record that a user has read and understood a specific version of an SOP.

**Active (SOP/Equipment)** — The current, operative status. Active SOPs are visible to all relevant users.

**Admin** — A flag that adds administrative capabilities (user management, department management, audit log access) to any user.

**Change Control (CC)** — A formal process for managing SOP updates. Requires manager signatures before the new version goes active.

**Department** — A team in the organisation. Determines which SOPs and equipment users see by default.

**Draft** — An SOP that has been created but not yet submitted for QA review.

**Manager** — A role that can submit SOPs, sign Change Controls, add equipment, and reassign PM tasks.

**Pending QA** — An SOP or equipment submitted for QA review, awaiting approval.

**PM (Preventive Maintenance)** — Scheduled maintenance tasks for equipment.

**Pulse** — The real-time notification feed showing what needs your attention.

**QA Manager** — A user in the QA department with Manager role. Has approval authority for SOPs and equipment.

**RLS (Row Level Security)** — Database-level security that controls what data users can access.

**Role** — Either Employee or Manager. Determines operational permissions.

**Secondary Department** — An additional department a user or asset belongs to, beyond the primary.
