The **Admin Console** is the centralized hub where administrative users can configure the platform, manage system-wide settings, and monitor operational health. This section of the knowledge base explains how to perform top-level tasks.

---

## Administrative Responsibilities

Unlike standard Managers or QA personnel who handle document workflows and equipment, **Admins** manage the infrastructure of the application itself.

Your primary responsibilities include:
- **User and Role Management**: Defining who has access to what, modifying roles, and granting or revoking administrative access.
- **Department Configuration**: Setting up the logical layout of the organisation's teams.
- **Compliance Monitoring**: Reviewing immutable Audit Logs, which track all significant security and data changes within the system.

## Navigation Options

The Admin features are accessed directly from the **Settings** menu. Use the sidebar to navigate to:

- `Users` - Manage accounts, roles, and admin grants.
- `Departments` - Create or rename operational units.
- `Audit Logs` - View strict system activity records.
